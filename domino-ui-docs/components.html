Project Structure

Any domino project is a maven multi module project and it contains two main modules that will be created by default, below lists the modules with an explanation for each,

  1. Front-End:
    • An aggregator for all front-end dependencies.

      • Explanation: responsible for collecting all front-end dependencies from other sub-modules.

    • Responsible for running the client side.

      • Explanation: compiles all collected dependencies to provide the final artifact that represents the client side of the application.

    • Can have the same application works on different front-end platforms.

      • Explanation: each platform for example desktop or web have a different main front-end module.

    Domino default frond-end implementation is GWT.

  2. Back-End:
    • An aggregator for all back-end dependencies.

      • Explanation: responsible for collecting all back-end dependencies from other sub-modules.

    • Responsible for running the server side.

      • Explanation: collects all dependencies to provide the final artifact that represents the server side of the application.

    • Can have the same application works on different back-ends.

      • Explanation: each back-end implementation for example vertx have a different main back-end module.

    Domino default back-end implementation is Vertx.



How do back-end and front-end communicate?

The communication between the back-end and front-end is only through restful calls, which means the modules do not depend on each others, you can have different front-ends and back-ends in different places and communicate between them through restful calls.
