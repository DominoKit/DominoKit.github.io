Quick Start

Domino is a simple very light Model-View-Presenter (MVP) framework for building enterprise applications on multiple platforms with the same code base. It supports the concept of separating applications into modules for easy and flexible use, and each module is tested separately which means your application is built in a modular and testable way.

Domino decouples the back-end from the front-end also decouples the front-end business logic from the presentation UI layer, you write your application as a separate maven module, moreover, each module is tested as a unit independently without the need to run the whole application, which means modules are independently testable.

Let's create your first domino project, steps below will guide you through the whole process it will only take you 5 minutes!

Create a Domino Project

Step One: Create the Project

Execute the below command, it will create a new domino project.

mvn archetype:generate \ -DarchetypeGroupId=com.progressoft.brix.domino.archetypes \ -DarchetypeArtifactId=domino-gwt-app-archetype \ -DarchetypeVersion=1.0-rc.2-SNAPSHOT

While executing, it will ask you to fill some values, please fill them as listed below:

  • groupId: com.foo
  • artifactId: sample
  • version: 1.0-SNAPSHOT
  • package: com.foo


You will realize that your project contains:

  • Two Main Folders: sample-backend, sample-frontend
  • Two Files: pom.xml, README.md

If you are using our current SNAPSHOTS builds, you will need to add the below profile to your setting.xml file,

<profile> <id>ossrh</id> <activation> <activeByDefault>true</activeByDefault> </activation> <repositories> <repository> <id>sonatype-snapshots-repo</id> <url>https://oss.sonatype.org/content/repositories/snapshots</url> <snapshots> <enabled>true</enabled> <updatePolicy>always</updatePolicy> <checksumPolicy>fail</checksumPolicy> </snapshots> </repository> </repositories> </profile>

Step Two: Build the Project

Make sure you are on the root directory, execute maven clean install as below command,


mvn clean install

Step Three: Run the Project

Execute the below command:


java -jar sample-backend/target/sample-backend-1.0-SNAPSHOT-fat.jar -conf ./sample-backend/target/classes/config.json

You can now access your application from the below URL:

localhost:8080

Check your browser console by clicking F12, if below logs appeared then your project is running successfully:


Sun Oct 29 15:59:42 GMT+200 2017 com.progressoft.brix.domino.gwt.client.Core
INFO: Initialize domino module...
Sun Oct 29 15:59:42 GMT+200 2017 com.progressoft.AppClientModule
INFO: sample Application frontend have been initialized.

Create your First Module

Step One: Create the Module

Execute the below command, it will create a new module, make sure you are on the root directory of the project that you had already created.

mvn archetype:generate \ -DarchetypeGroupId=com.progressoft.brix.domino.archetypes \ -DarchetypeArtifactId=domino-gwt-module-archetype \ -DarchetypeVersion=1.0-rc.2-SNAPSHOT \ -DarchetypeParentGroupId=com.foo \ -DarchetypeParentArtifactId=sample \ -DarchetypeParentVersion=1.0-SNAPSHOT

While executing, it will ask you to fill some values, please fill them as listed below:

  • groupId: com.foo
  • artifactId: hello-world
  • version: 1.0-SNAPSHOT
  • package: com.foo
  • module: Test
  • subpackage: helloworld


You will realize that your module is created inside your previous created project, it will contain:

  • Four Main Folders: hello-world-backend, hello-world-frontend, hello-world-frontend-ui, hello-world-shared
  • One File: pom.xml

Step Two: Build the Module

You have to add the module dependencies in your main project, all what you have to do is to add the below dependencies in /sample/sample-frontend/pom.xml,

<!-- frontend dependency --> <dependency> <groupId>com.foo</groupId> <artifactId>hello-world-frontend</artifactId> <version>1.0-SNAPSHOT</version> </dependency> <!-- frontend UI dependency --> <dependency> <groupId>com.foo</groupId> <artifactId>hello-world-frontend-ui</artifactId> <version>1.0-SNAPSHOT</version> </dependency> <!-- shared dependency --> <dependency> <groupId>com.foo</groupId> <artifactId>hello-world-shared</artifactId> <version>1.0-SNAPSHOT</version> </dependency>

Make sure you are on the root directory, execute maven clean install as below command,

mvn clean install

Step Three: Run the Project with your Module

Follow the same steps above in order to run your project with your module.

Now check your browser console by clicking F12, you will realize more logs will appear as shown below:

IDE Integration

You can use our run configurations in order to help you through your development cycle, you can find the run configurations folder in your root directory.

IntelliJ

Below steps will guide you through implementing our run configurations in IntelliJ,

  1. Import your project to IntelliJ.
  2. Copy the run configurations folder into your .idea.

Note that now you can find your run configurations in IntelliJ Select Run/Debug Configuration tab.


Below lists a brief description on our run configurations,

Testing mode:
  • ALL_TESTS: runs all test cases in your project.

Development mode:
  • process_sources: copies all your resource files into your target.
  • sample.xml: production run (after mvn clean install).

Super dev mode:
  • sample-dev: start your application in a dev mode.
  • super_dev_mode: start gwt super dev mode.
